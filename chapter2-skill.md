## 🔥 Skill 是什么？一个职场比喻秒懂

> **新员工入职** → 公司会安排"入职培训"
> - 通用培训：企业文化、规章制度 → 就像 OpenClaw 的**基础能力**
> - 技能培训：Excel、PPT、工具使用 → 就像 **Skill（技能）**
> - 业务培训：产品知识、业务流程 → 就像 **行业知识库**

**没有培训的新员工 = 没有Skill的AI**
→ 只能做最基础的事，专业工作干不了

---

## 一、Skill = AI 的"岗前培训包"

| 职场概念 | 对应 OpenClaw |
|---------|---------------|
| 员工技能培训 | Skill |
| 培训课件 | SKILL.md |
| 培训后能上手的工作 | Skill 能解决的问题 |
| 培训老师 | 写 Skill 的人 |
| 员工手册 | references/ 参考文档 |

### 为什么要用 Skill？

1. **省钱**：不用每个任务都从头教 AI
2. **高效**：培训一次，终身可用
3. **专业**：普通人需要 2 小时学的，AI 培训后 2 秒搞定
4. **标准化**：像公司 SOP 一样，输出稳定

## 二、Skill 解决什么问题？

| 痛点 | 没有 Skill | 有 Skill |

### 👉 创业者痛点

| 痛点 | 没有 Skill | 有 Skill |
|------|-----------|----------|
| 客服回复 | 每次手动敲字 | 自动按话术库回复 |

| 数据分析 | 每周手动汇总 | 自动生成报表 |

## 五、Skill 的技术结构

### 📦 像搭积木一样

```
我的 Skill/
├── SKILL.md           # 说明书（必需）
│   ├── 这个技能是干啥的？
│   ├── 什么时候用？
│   └── 怎么用？
├── scripts/           # 工具箱（可选）
│   └── 自动化脚本
├── references/       # 知识库（可选）
│   └── 行业资料/模板
└── assets/           # 素材库（可选）
    └── 图片/模板文件
```

### 📝 SKILL.md 核心格式

```markdown
---
name: 周报生成器
description: "每周一自动生成周报。员工说'写周报'、'周报好了'时自动触发。"
metadata: { "openclaw": { "emoji": "📊" } }
---

# 周报生成器使用指南

## 什么时候用？
- 每周一需要写周报
- 不知道周报怎么写
- 想按照公司模板写

## 怎么说？
- "帮我写个周报"
- "本周工作总结"
- "生成上周工作汇总"

## 输出格式
```
## 本周工作
-

## 下周计划
-

## 遇到的问题
-
```
```

## 七、Skill 的加载优先级

| 优先级 | 来源 | 路径 |
|--------|------|------|
| 🥇 高 | Workspace（你当前的） | `<workspace>/skills` |
| 🥈 中 | 你自己安装的 | `~/.openclaw/skills` |
| 🥉 低 | 系统内置 | 安装包自带 |

> 就像公司制度：**先看公司章程 → 再看部门规定 → 最后参考国家标准**

## 八、如何开始？

### 🚀 立即可用

```bash
# 1. 查看已安装的技能
openclaw skills list

# 2. 安装一个新技能
clawhub install weather

# 3. 直接开聊
# "北京今天天气怎么样？"
```

### 📚 学习路径

| 阶段 | 目标 | 建议 |
|------|------|------|
| 小白 | 会用现有 Skill | 每天试 1-2 个 |
| 进阶 | 会改 Skill 配置 | 尝试修改描述 |
| 熟练 | 会写简单 Skill | 从自己工作流开始 |
| 专业 | 能卖 Skill 赚钱 | 做一个垂直领域的 Skill 包 |
