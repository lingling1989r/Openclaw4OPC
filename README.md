## 📖 课程大纲

### 第一章：认知与案例

**【导读】**

接触一个新事物，首先要做的事是什么？

**是先看别人怎么用。**

工具是同一个工具，但使用工具的人不同、水平不同，使用出来的效果也千差万别。就像定海神针，在普通人手里可能就是个铁棒，但在孙悟空手里，就是一根可伸可缩、打遍天下的金箍棒。

所以，我们要去看看那些厉害的人是怎么使用这个工具的。

**学习目标：** 了解 OpenClaw 能做什么，看别人怎么用

**内容包括：**
- 1.1 OpenClaw 是什么？一个可以帮你打造个人 AI 助理的工具
- 1.2 案例分享：别人怎么用 OpenClow 解决实际问题
- 1.3 对照自己所在的行业，去看看自己行业的案例
- 1.4 工具的边界与可能性：理论边界 vs 实际边界

**你将获得：**
- 对 OpenClaw 的基本认知
- 行业应用灵感
- 案例参考素材

### 第三章：需求梳理

**【导读】**

"我不想学这个东西，我能不能告诉你我的需求，付费请你帮我定制一个出来？"

这是大部分人会有的一个想法。但我第一反应碰到这类需求，其实不太愿意接。因为我接了太多项目，实在知道大家最大的问题根本不是实现上，而是在需求理不清楚、需求会变动的事情上。

**为什么会这样？**

第一，在有这个工具之前——就像你没有用过拖拉机之前，你的工作方式，和你用了拖拉机之后，工作方式的变化是不一样的。你根本没有梳理过工作流，想干嘛干嘛，想起啥就做啥，你不会去想批量处理（batch）的工作方式是怎么样子的。

第二，对 AI 的边界不了解，完全不知道它能做成啥样。

第三，你脑子里想的、别人说给你的，和最后交付的差距是没有概念的。你根本都没有看过，所以你完全不知道最后使用它的时候，实际用起来又是什么样的一个效果。

还有一个原因就是畏难情绪。因为大家不了解，所以就想，我把这个事情外包出去，是不是就解决了。但是最后用的还是你，你还是得考虑顺不顺手。

**按照我的方法，梳理完需求之后有两个好处：**

1. 你去找团队时，报价都能砍半价。因为对方在成本核算时，已经为不明确的需求预留了一半的预算。
2. 你知道你要什么了。不管你有没有现成的工作流，这都会成为你对当前工作的一次沉淀和资产，它会变成一件有复利价值的事情。

**学习目标：** 学会清晰表达需求

**内容包括：**
- 3.1 为什么要梳理需求？
- 3.2 宏观行业需求分析：以终为始，了解行业整体需求
- 3.3 微观个人需求梳理：盘点过去的工作流
- 3.4 工作流分析与优化

**你将获得：**
- 需求梳理方法
- 工作流优化思路
- 外包沟通能力

### 第五章：从超级个体到一人公司

**【导读】**

外包模式是指有临时性的需求时，去找外部资源完成。而组织模式则是可管理的公司模式。

当它成为一个长期需求时，我们就需要固定的分工和职责。等工作量上来后，需要协同配合，我们就需要约定好，让它能自动运行，开始有协同。所以你要开始规范化的梳理。

我会以我自己的案例来讲解这个过程。

剩下的部分，因为行动营在讲 Clubhouse 的获客，那我的获客部分，我要给大家拆解我的三大业务，把我的三大业务梳理出来。

在这之前，我最好给自己定一个行动量。我最好在此之前，把我的 OPC 研发团队也搞进来，因为我需要我的研发先上线。

**学习目标：** 从单兵作战到规范化、规模化

**内容包括：**
- 5.1 什么是超级个体？一个人就是一支团队
- 5.2 什么时候需要"一人公司"？长期需求的处理
- 5.3 团队协作规范：从外包到自建
- 5.4 自动化流程设计：让系统帮你运行
- 5.5 效率提升方案：协同与分工

**你将获得：**
- 规模化思维
- 团队管理方法
- 系统化能力
