## 6. 赚钱思路：先借大势，再找需求

赚钱思路是啥？就是我们要去找需求。

在每一个新事物出来的时候，都伴随着新的诉求。

### 6.1 第一件事：借大势（破除心魔）

首先，你要借大势，不能顾影自怜，或者说固步自封，守着之前的那个业务不肯放——这其实是很大机会。

我身边的感受是：

- 对个人来讲还好
- 尤其是企业老板，最容易有这个顾忌
- 大家都始终想要推自己的产品

### 6.2 第二件事：和自己的过往经验结合（决定你赚什么钱）

破除了第一个心魔后，下面我们就来谈：

- 怎么去赚钱
- 怎么和自己的过往经验结合
- 怎么和咱们已有的东西结合

因为如果人人都能赚钱，但每个人赚的肯定是不一样的钱。

比较核心的还是：**你自己**。

---

## 7. 三条变现路线（从第一波钱到可持续）

### 7.1 路线 1：安装部署（第一波钱，量大）

一个很大的痛点是：**安装部署**。

很多人确实解决不了这个问题。

比如“农家乐养龙虾”这种场景：

- 它客单价没那么高
- 但量大
- 非常有用

我分享一个身边小伙伴的例子：他的挣钱方式比较符合大众的预期，属于赚第一波钱的。

### 7.2 路线 2：Skill（小范围业务的直接变现）

很多人不知道 Skill 怎么挣钱。

如果你真的有一个非常成熟的 Skill：

- 在 CloudHub 或 GitHub 上装 Skill 是免费的
- 但说实话，很多业务大家是不愿意公开的
- 它一定是小范围的

所以 **Skill 这一块是可以直接去变现的**。

### 7.3 路线 3：从“手停脚停”到“产品化/广场化/持续收益”

刚刚说的是一对一的定制化。

也可以有更通用的方式，类似于 SaaS 产品。

对于前面那种方式，它是手停脚停的。

对我自己来讲，我会去盘算，我希望要的是……

> 首先，这肯定需要一个过程。你肯定要先经历第一个阶段，才能走到第二个阶段。
>
> 如果你有现成的业务，那就可以直接走第二条路线，根本不需要走第一条路线。为什么？因为现在是 Web Coding 的时代，我们就可以走这个。
>
> 在第二条路线里，我们就需要有 Skill，然后去推广它。
>
> 第三条路线，就是放到 Skill 广场里面去变现。它的好处是，做一套就可以持续产生收益。
>
> 那第三条路线的话，就是假设我们已经有自己成熟的产品，怎么去变现？收入、商品怎么变现？大面上来讲，就是成熟产品的变现方式。
>
> 成熟产品变现的方式，可以和……（这里可以借鉴楼下的模式）。我们所有人都在做类似的事情。我们可以在 GitHub 里面去处理，而且它还有一个好处，能够节省我们自己服务器的成本。付费是在这一端的。这个架构可以给大家画一下。

## 8. 结尾：一句话总结

- **借大势**：别守着旧业务不肯放
- **找需求**：新事物带来新诉求
- **结合自己**：你决定你赚什么钱
- **从 1 到 3**：先赚第一波钱，再走向可持续
