## 1.1 OpenClaw 是什么？

> 一个可以帮你打造个人 AI 助理的工具

OpenClaw（俗称"大龙虾"）是一个可以在你自有设备上运行的个人 AI 助手。

### 核心能力

| 能力 | 说明 |
|------|------|
| 多渠道接入 | 飞书、微信、Telegram、Discord、Slack 等 20+ 渠道 |
| 本地运行 | 数据存在自己电脑上，隐私安全 |
| 技能系统 | 每天一个 SKILL，持续进化 |
| 多 Agent | 可以同时运行多个 AI"员工" |
| 语音交互 | 支持语音唤醒、语音对话 |

### 为什么叫"龙虾"？

因为它的 Logo 是小龙虾，社群都叫它"大龙虾"或"小龙虾"。

## 1.1.1 OpenClaw vs 豆包：有什么区别？

> 很多人问：OpenClaw 和字节跳动的"豆包"AI 助手有什么区别？

### 核心差异对比

| 维度 | OpenClaw（龙虾） | 豆包（字节） |
|------|------------------|--------------|
| **运行方式** | 本地部署，数据在你自己设备 | 云端服务，数据在字节服务器 |
| **定制程度** | 高度可定制，想让它做什么就做什么 | 功能固定，按产品设定走 |
| **数据隐私** | 数据本地存储，隐私安全 | 数据上传云端 |
| **使用门槛** | 需要部署配置 | 下载即用 |
| **渠道** | 飞书/Telegram/微信/Discord 等 | 字节生态（抖音/飞书等） |
| **价格** | 免费（需自备模型 API） | 免费 |
| **适用人群** | 开发者/技术爱好者/有定制需求 | 普通用户 |

### 一句话总结

| 产品 | 特点 |
|------|------|
| **豆包** | 你适应工具——下载 app，按照产品设定的方式使用 |
| **OpenClaw** | 工具适应你——让它做什么，它就做什么 |

### 比喻

- **豆包** 像是租房子——设施齐全，拎包入住，但不能随便改动
- **OpenClaw** 像是自己盖房子——要从头搭建，但想怎么建就怎么建

## 1.1.3 认知跃迁：从"咨询台"到"请到家里"

> 理解这个差别，你就知道为什么要用 OpenClaw

### 豆包 vs OpenClaw：一句话讲清楚

| 产品 | 比喻 | 说明 |
|------|------|------|
| **豆包** | 咨询台 | 每天跑到别人家门口问问题 |
| **OpenClaw** | 请到家里 | 直接让它帮你干活 |

> **现在的 OpenClaw，更像是你把它请到家里，让它直接帮你去做，比如打扫房间。你不用问它怎么做，只需要告诉它你想干什么，它就会帮你把事情做了。**

### 为什么会差别这么大？

**核心差别：能不能"执行"**

| 维度 | 豆包/传统AI | OpenClaw |
|------|-------------|----------|
| 你做什么 | 问问题，等答案 | 告诉它目标，它去执行 |
| 信息获取 | 你接收信息再操作 | 它直接操作你的电脑 |
| 执行方式 | 你动手 | 它动手，你动嘴 |

### "通道"才是杀手锏

很多人觉得 OpenClaw 技术含量不高，但它的"通道"概念才是用户体验的核心：

| 之前 | 现在 |
|------|------|
| 跑到电脑前操作 | 躺着说话就能办事 |
| 刷剧时无法干活 | 动嘴指挥它 |
| 带娃没时间工作 | 说话间事就办完了 |

> **现在变成是拿着手机，说说话，动动嘴，就完全实现了"用嘴巴干活"的理想生活。**

### 真实使用场景

| 场景 | 以前 | 现在 |
|------|------|------|
| 带娃 | 没办法用电脑 | 说话就能处理工作 |
| 坐火车 | 带着电脑才安心 | 手机指挥服务器 |
| 户外 | 断联焦虑 | 随时安排任务 |
| 生病住院 | 躺床上干着急 | 动嘴指挥"龙虾" |

## 1.1.4 部署选择：就近原则 vs 源头原则

> 网上资料鱼龙混杂，怎么选？

### 两原则

| 原则 | 说明 | 推荐 |
|------|------|------|
| **就近原则** ❌ | 搜索排第一的就用 | 不推荐 |
| **源头原则** ✅ | 只看官方 | **推荐** |

### 市面三大类（别混淆！）

1. **OpenClaw 官方** — 源头
2. **各种 Cloud** — 域名抢注（营销策略）
3. **魔改版** — 非官方

> OpenClaw 火起来后，大家干的第一件事是疯狂买 .cloud 域名。这确实是好策略，但作为用户，别混淆了。

### 三种部署方式对比

| 方式 | 难度 | 扩展性 | 推荐场景 |
|------|------|--------|----------|
| **官方源码部署** | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 有技术基础，想深度定制 |
| **云服务一键部署** | ⭐⭐ | ⭐⭐⭐ | 想先体验 |
| **App/第三方** | ⭐ | ⭐ | 纯小白，只想用起来 |

### 我的选择建议

| 人群 | 推荐方式 | 理由 |
|------|----------|------|
| 有技术基础 | 服务器+官方源码 | 扩展性强，功能全 |
| 想先体验 | 云服务器 | 门槛低，99元/年 |
| 纯小白 | App/第三方 | 下载即用 |

> **你吃不了前面的苦，就得吃后面的苦。**
> 
> 前期省事，后期受限；前期吃苦，后期自由。

### 成本参考

| 方案 | 成本 | 说明 |
|------|------|------|
| 云服务器 | 99元/年 | 阿里云/腾讯云引流款 |
| 大模型 API | 几十元/月 | MiniMax 等 |
| Mac mini | 500元+ | 一次性投入 |

> **一个月几十块钱，也就是一杯奶茶钱。**

## 1.1.5 傅盛案例拆解：明线与暗线

> 傅盛的"龙虾养成日记"是 OpenClaw 传播最广的案例，值得深度拆解。

### 明线：龙虾怎么用

| 案例 | 说明 | 我们能学什么 |
|------|------|--------------|
| 8个AI"同事"协同 | 不同技能分工合作 | AI 可以同时分担多种工作 |
| 48小时139技能 | 快速验证新技能 | 低成本试错 |
| 健康管家 | 监督康复日程 | 个人知识管理/提醒助手 |
| 交易官 | 7×24小时分析 | 自动化数据分析 |
| 初中生都能用 | 降低心理障碍 | 告诉用户"门槛很低" |

### 暗线：营销手法拆解

> 傅盛为什么能让这个案例全网传播？

| 暗线手法 | 目的 | 我们能借鉴什么 |
|----------|------|---------------|
| 讲故事 > 讲功能 | 真实故事更有说服力 | 用案例代替功能表 |
| 现场演示 > 列参数 | 眼见为实 | 现场演示更有冲击力 |
| "50块139技能" | 低成本吸引力 | 强调投入产出比 |
| 引导加群 | 私域沉淀 | 引流到持续触达的地方 |
| 初中生都能用 | 降低心理障碍 | 告诉用户"门槛很低" |
| 线上+线下结合 | 建立忠实用户 | "按头"让你用起来 |

### 暗线背后的逻辑

1. **降低门槛**：先让你用起来，续费率才是关键
2. **私域运营**：加群是为了帮你解决问题，提高留存
3. **夹带私货**：最后一页PPT是人工写的，核心是推广自己的产品

> **金句：** "产品引流到位了，但续费率不高是个大问题。"

## 1.3 AI 的边界：理论边界 vs 实际边界

### 两种边界

| 边界类型 | 解释 | 示例 |
|----------|------|------|
| **理论边界** | 技术上 AI 能做什么 | 理论上可以用 AI 做任何事 |
| **实际边界** | 市场上别人验证过能做什么 | 别人真的跑通并拿到结果的 |

### 为什么实际边界更重要？

- **理论可行 ≠ 实际能做**
- **有人做过 ≠ 你能做**
- **做的人多 ≠ 适合你**

> **金句：** "别人没跑过的领域，先让子弹飞一会儿。"
> 
> **行动：** 先看案例，再决定要不要动手。

## 1.4 我们的边界

### 什么是"我们的边界"？

| 问题 | 答案 |
|------|------|
| 哪些事我想放权给 AI？ | 重复性劳动、信息整理、数据分析 |
| 哪些事我不想交给 AI？ | 核心决策、人情世故、需要创造力的事 |
| 我的核心能力是什么？ | 判断力、审美、战略思考 |

### 为什么要想"边界"？

1. **效率最大化** — 把重复的事交给 AI，专注高价值的事
2. **不踩坑** — 知道 AI 擅长什么、不擅长什么
3. **心理舒适** — 明确哪些事必须自己掌控

## 1.10 龙虾部署：5分钟上手指南

> 很多人被"部署"这个词吓到了，其实真的没那么难。

### 最低成本方案

| 方案 | 设备 | 成本 | 适合场景 |
|------|------|------|----------|
| **废弃手机** | 旧 Android 手机 | 0元 | 轻度使用（消息处理） |
| **二手 Mac mini** | 二手平台 ~500元 | 500元 | 主流推荐，长期运行 |
| **NAS 部署** | 群晖/威联通 | 2000元+ | 开发者/多用户 |
| **云服务器** | VPS | 50元/月 | 需要外网访问 |

### 环境要求

| 要求 | 说明 |
|------|------|
| **Node.js** | 需要 22 或更高版本 |
| **系统** | macOS、Linux、Windows (WSL2) |
| **网络** | 能访问 OpenAI/Claude 等模型 API |

### 一键部署（5分钟）

```bash
# Step 1: 安装（Mac/Linux）
curl -fsSL https://openclaw.ai/install.sh | bash

# Step 2: 初始化向导
openclaw onboard --install-daemon

# Step 3: 检查状态
openclaw gateway status

# Step 4: 打开控制台
openclaw dashboard
```

**Windows 用户:**
```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### 部署后第一天该做什么？

| 时间 | 动作 | 说明 |
|------|------|------|
| 第1天 | 配置一个 SKILL | 从技能市场安装一个，比如天气查询 |
| 第2天 | 设置定时任务 | 每天早上自动推送热点资讯 |
| 第3天 | 接入一个渠道 | 飞书或Telegram，先测试对话 |
| 第4天 | 记录使用场景 | 连续3天记录用它做了什么 |

### 常见问题速查

| 问题 | 解决方案 |
|------|----------|
| 端口被占用 | `openclaw gateway start --port 9091` |
| 模型连不上 | 检查 API Key 配置是否正确 |
| 消息发不出去 | 确认渠道配置是否完成 |
| 响应很慢 | 确认网络环境，或换用国内模型 |

## 1.12 盘清楚自己的需求

> 工具是为了解决问题，而不是为了学工具。

### 需求自检表

回答这些问题，找到你的切入点：

| 问题 | 回答方向 |
|------|----------|
| 你每天重复做什么？ | 复制粘贴、整理数据、回复同样问题 |
| 你经常忘记什么事？ | 定期检查、跟进提醒 |
| 你想自动化什么？ | 定时推送、自动回复、数据抓取 |
| 你的信息源有哪些？ | 公众号、知乎、Twitter、邮件 |

### 从需求到技能

| 需求 | 对应技能 |
|------|----------|
| 每天早上自动发热点 | 定时任务 + web_fetch |
| 自动回复客户常见问题 | 知识库 + 自动回复 |
| 监控竞品价格变化 | 浏览器自动化 + 告警 |
| 自动生成周报 | 定时任务 + AI 总结 |
| 多平台内容一键分发 | 浏览器自动化 |

### 行动建议

1. **只盘 1 个需求** — 多了顾不上
2. **只跑通 1 个技能** — 跑通后再扩展
3. **记录使用数据** — 知道它为你省了多少时间
