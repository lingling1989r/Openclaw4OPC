## 1.1 OpenClaw 是什么？

> 一个可以帮你打造个人 AI 助理的工具

OpenClaw（俗称"大龙虾"）是一个可以在你自有设备上运行的个人 AI 助手。

### 核心能力

| 能力 | 说明 |
|------|------|
| 多渠道接入 | 飞书、微信、Telegram、Discord、Slack 等 |
| 本地运行 | 数据存在自己电脑上，隐私安全 |
| 技能系统 | 每天一个 SKILL，持续进化 |
| 多 Agent | 可以同时运行多个 AI"员工" |

## 1.2 认知框架：不做什么比做什么更重要

### 三个核心认知

| 认知 | 解释 | 行动 |
|------|------|------|
| **不做什么** | 避免踩坑，避免做回报低的事 | 别人没跑过的领域，先让子弹飞一会儿 |

| **盘需求** | 知道自己要什么，比学工具更重要 | 先想清楚要解决什么问题 |

### 为什么要"让子弹飞"？

> **案例：** 有人问"微信能做吗？"
> 
> 如果你不是从事这一行的，别人没跑过的领域，先观望。
> 
> 市场水很深，不是每个想法都能落地。
> 
> **结论：** 盘需求，赢了一大半。

## 1.3 AI 的边界：理论边界 vs 实际边界

### 两种边界

| 边界类型 | 解释 | 示例 |
|----------|------|------|
| **理论边界** | 技术上 AI 能做什么 | 理论上可以用 AI 做任何事 |
| **实际边界** | 市场上别人验证过能做什么 | 别人真的跑通并拿到结果的 |

### 为什么实际边界更重要？

- **理论可行 ≠ 实际能做**
- **有人做过 ≠ 你能做**
- **做的人多 ≠ 适合你**

> **金句：** "别人没跑过的领域，先让子弹飞一会儿。"
> 
> **行动：** 先看案例，再决定要不要动手。

## 1.4 我们的边界

### 什么是"我们的边界"？

| 问题 | 答案 |
|------|------|
| 哪些事我想放权给 AI？ | 重复性劳动、信息整理、数据分析 |
| 哪些事我不想交给 AI？ | 核心决策、人情世故、需要创造力的事 |
| 我的核心能力是什么？ | 判断力、审美、战略思考 |

### 为什么要想"边界"？

1. **效率最大化** — 把重复的事交给 AI，专注高价值的事
2. **不踩坑** — 知道 AI 擅长什么、不擅长什么
3. **心理舒适** — 明确哪些事必须自己掌控

## 1.5 龙虾 vs 传统 APP：最大的差异

### 传统 APP

| 特点 | 说明 |
|------|------|
| 功能固定 | 做什么是开发好的 |
| 门槛固定 | 会用就会用，不会用还是不会 |
| 迭代慢 | 功能更新需要开发周期 |

### 龙虾（OpenClaw）

| 特点 | 说明 |
|------|------|
| **可定制** | 你让它做什么，它就做什么 |
| **可学习** | 每天一个 SKILL，持续进化 |
| **可成长** | 越用越懂你，越用越顺手 |

### 最大的差异

> **传统 APP：** 你适应工具  
> **龙虾：** 工具适应你

## 1.10 龙虾部署：低成本起步指南

> 很多人被"部署"这个词吓到了，其实50块就能跑起来。

### 最低成本方案（50元）

| 方案 | 设备 | 成本 | 适合场景 |
|------|------|------|----------|
| **废弃手机** | 旧 Android 手机 | 0元 | 轻度使用（消息处理） |
| **二手 Mac mini** | 二手平台 ~500元 | 500元 | 主流推荐 |
| **NAS 部署** | 群晖/威联通 | 2000元+ | 长期运行、多用户 |

### 一键部署流程（5分钟）

```bash
# 方式1：Mac 用户（推荐）
brew install openclaw
openclaw gateway start

# 方式2：Docker（跨平台）
docker run -d -p 9090:9090 openclaw/openclaw

# 方式3：直接下载
curl -sL https://get.openclaw.ai | bash
```

### 部署检查清单

| 步骤 | 操作 | 预计时间 |
|------|------|----------|
| 1 | 安装 Node.js (18+) | 2分钟 |
| 2 | 安装 OpenClaw | 1分钟 |
| 3 | 配置渠道（飞书/Telegram） | 2分钟 |
| 4 | 添加第一个 SKILL | 1分钟 |
| 5 | 测试对话 | 1分钟 |

### 常见部署问题

| 问题 | 解决方案 |
|------|----------|
| 端口被占用 | 改用其他端口：`openclaw gateway start --port 9091` |
| 模型连不上 | 检查 API Key 配置（支持 OpenAI / Claude / 国产模型） |
| 消息发不出去 | 检查渠道配置是否正确 |
| 响应很慢 | 确认网络环境，或换用国内模型 |

### 部署后第一天该做什么？

1. **配置一个SKILL** — 比如天气查询、热点追踪
2. **设置一个定时任务** — 每天早上推送热门资讯
3. **接入一个渠道** — 先飞书或Telegram测试
4. **记下使用场景** — 连续3天记录用它做了什么

## 1.12 盘清楚自己的需求

> 工具是为了解决问题，而不是为了学工具。

### 需求自检表

回答这些问题，找到你的切入点：

| 问题 | 回答方向 |
|------|----------|
| 你每天重复做什么？ | 复制粘贴、整理数据、回复同样问题 |
| 你经常忘记什么事？ | 定期检查、跟进提醒 |
| 你想自动化什么？ | 定时推送、自动回复、数据抓取 |
| 你的信息源有哪些？ | 公众号、知乎、Twitter、邮件 |

### 从需求到技能

| 需求 | 对应技能 |
|------|----------|
| 每天早上自动发热点 | 定时任务 + web_fetch |
| 自动回复客户常见问题 | 知识库 + 自动回复 |
| 监控竞品价格变化 | 浏览器自动化 + 告警 |
| 自动生成周报 | 定时任务 + AI 总结 |
| 多平台内容一键分发 | 浏览器自动化 |

### 行动建议

1. **只盘 1 个需求** — 多了顾不上
2. **只跑通 1 个技能** — 跑通后再扩展
3. **记录使用数据** — 知道它为你省了多少时间
