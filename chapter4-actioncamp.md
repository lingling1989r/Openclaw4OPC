## 本章目录

## 4.1 理念纠正：AI 是员工，不是外包

> AI 是长期协作的"数字员工"，不是一次性外包

### ❌ 错误理解：AI 是外包

很多人把 AI 当作"外包"，这是错误的。

| 外包 | AI |
|------|-----|
| 一次性项目 | 长期协作 |
| 做完就结束 | 持续工作 |
| 需要详细沟通 | 说清楚就能干 |

### ✅ 正确理解：AI 是员工

AI 是要和你**长期协作**的数字员工。

| 员工 | AI |
|------|-----|
| 发工资才能干活 | 给 API 就能干活 |
| 需要培训 | 需要提示词 |
| 会请假会离职 | 7×24 小时待命 |
| 成本高 | 成本极低 |

## 4.2 大SOP：人机协作流程

> 我们跟机器协作的 SOP

### 什么是大SOP？

**人机协作的工作流程**——什么时候人做？什么时候机器做？机器干完后，谁来接手？

### 核心问题

| 问题 | 回答 |
|------|------|
| 什么时候该人做？ | 需要判断力、情感、决策的事 |
| 什么时候该机器做？ | 重复性、信息整理、数据处理 |
| 机器干完后，谁来接手？ | 需要审核/决策的环节 |
| 怎么检查机器的输出？ | 设置检查节点 |

### 举例：大SOP

```
需求：每天早上自动发行业热点

大SOP流程：
① 机器抓取热点 → ② 人审核把关 → ③ 机器分发到群 → ④ 人处理反馈
                 ↑检查点           ↑交接点
```

### 大SOP梳理步骤

| 步骤 | 动作 |
|------|------|
| 1 | 列出任务全流程 |
| 2 | 标注每个环节的负责人（人/机器） |
| 3 | 明确交接点（机器→人 或 人→机器） |
| 4 | 设置检查节点 |

## 4.3 小SOP：机器内部的SOP

> 机器自己的 SOP，核心是输入 + 输出

### ❌ 常见误区

以前很多人觉得，梳理小SOP就是要把中间流程也画得很详细——其实不需要。

**中间流程哪怕给不到都没关系，AI 会自己处理。**

### ✅ 正确姿势：输入 + 输出

梳理小SOP时，最重要的是约定好**输入**和**输出**：

| 要素 | 说明 |
|------|------|
| **输入** | 这个任务需要什么材料/信息？（来自现实世界） |
| **输出** | 期望产出什么结果？ |

### 举例：小SOP

| 任务 | 输入 | 输出 |
|------|------|------|
| 写周报 | 本周工作记录 | 按模板生成的周报 |
| 查天气 | 城市名称 | 天气情况+建议 |
| 搜集资料 | 关键词 | 整理好的资料汇总 |
| 写文案 | 产品信息+风格要求 | 完成的文案 |
| 整理会议纪要 | 会议录音/记录 | 结构化的会议纪要 |

### 小SOP梳理步骤

| 步骤 | 动作 |
|------|------|
| 1 | 列出工作中日常会做的任务 |
| 2 | 明确每个任务的**输入**（你需要给AI什么） |
| 3 | 明确每个任务的**输出**（AI应该产出什么） |
| 4 | 测试验证 |

## 4.4 员工模式的适用场景

### 什么样的任务适合交给"AI员工"？

| 场景 | 特点 | 适合度 |
|------|------|--------|
| 重复性工作 | 每天/每周都要做 | ⭐⭐⭐⭐⭐ |
| 信息整理 | 收集、汇总、分类 | ⭐⭐⭐⭐⭐ |
| 内容生成 | 文案、报告、邮件 | ⭐⭐⭐⭐ |
| 数据分析 | 统计、汇总、趋势 | ⭐⭐⭐⭐ |
| 客服回复 | 常见问题标准化回答 | ⭐⭐⭐⭐ |

### 什么样的任务不适合交给AI？

| 场景 | 原因 |
|------|------|
| 复杂决策 | 需要行业经验判断 |
| 情感沟通 | 需要人情世故 |
| 核心创意 | 需要独特审美判断 |
| 需要信任关系的合作 | 对外沟通仍需真人 |

## 4.5 实战：梳理你的"员工"任务

### 练习1：列出你的日常任务

**步骤：**
1. 列出你每天/每周要做的所有任务
2. 逐个判断：这个任务能否交给 AI 做？
3. 选出 Top 3，验证输入+输出

### 练习2：写一个任务卡

**模板：**

```
任务名称：[例如：写周报]
输入：我需要给AI什么 → [本周工作记录、模板要求]
输出：AI应该产出什么 → [按模板生成的周报]
验证：实际用一下，看输出是否符合预期
```

### 常见可AI化的任务

| 任务 | 输入 | 输出 |
|------|------|------|
| 📧 邮件回复模板 | 客户问题+回复要点 | 完整的邮件 |
| 📊 周报/数据汇总 | 原始数据 | 整理好的周报 |
| 📝 会议纪要 | 会议记录 | 结构化纪要 |
| 🔍 竞品资料搜集 | 竞品名称+关注点 | 资料汇总 |
| 📝 社媒内容 | 产品信息+风格要求 | 完成的文案 |

## 4.6 成果验收与反馈

### 验收标准

完成练习后，请检查：

| 检查项 | 合格标准 |
|-------|---------|
| 输入清晰 | 知道需要给AI什么材料 |
| 输出明确 | 知道期望产出什么 |
| 测试可用 | 实际跑通，有可用输出 |

### 反馈机制

**每日复盘问题：**
1. 今天交给AI做的任务，成功率是多少？
2. 哪些任务AI做得很好？哪些不行？
3. 输入+输出约定清楚了，AI表现是否稳定？
